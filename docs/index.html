Portfolio project · retail personalization

Behavior-aware ranking with RecSys Challenge 2025

A production-style ML/data engineering project that turns anonymized online-retailer event logs into user profiles, personalized product rankings, and comparable recommendation experiments.

View repository Architecture Demo design

Why RecSys 2025?

ESCI is excellent for general query-product relevance, but it does not contain user identities or preference histories. RecSys Challenge 2025 is a better fit for personalization because it includes behavioral event streams: purchases, add-to-cart, remove-from-cart, page visits, search queries, product attributes, and supervised targets.

System Architecture

Data layer

Polars reads parquet event logs and target files. ClickHouse is the local analytical store for events, features, predictions, and evaluation runs.

Modeling layer

Start with transparent personalized heuristics, then compare popularity, co-occurrence, BPR/matrix factorization, and gradient-boosted propensity rankers.

Serving layer

FastAPI serves a local UI where a user can assume validation-set personas and see how ranking changes with inferred preferences.

Interactive Demo

The local demo exposes anonymized validation users as human-readable roles such as Decisive mid-price category hopper. Product names and queries are anonymized numeric-token strings, so the UI supports filters like category:6059, sku:1073042, or encoded query tokens.

Evaluation Plan

Ranking quality

MAP@K and Recall@K over validation target products, plus category and price-bucket diagnostics.

Behavioral learning

Compare static popularity against user-history-aware models to quantify personalization lift.

Engineering quality

Track training time, inference latency, reproducibility, experiment artifacts, and local deployment boundaries.

What Is Production-Like?

The repo uses a package structure, Docker Compose, typed Python modules, Polars data processing, ClickHouse schemas, MLflow-ready experiment tracking, FastAPI serving, pytest coverage, and GitHub Actions CI. The hosted page is static; the data services run locally or on a separate deployment target.

Dataset: Synerise RecSys Challenge 2025. This project uses anonymized event data and does not expose raw product names or customer identities.
