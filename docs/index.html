View repository

Why This Dataset?

Unlike search relevance datasets that only judge query-product pairs, RecSys 2025 contains user behavior over time: purchases, carts, removals, page visits, search queries, product metadata, and validation targets. That makes it a better foundation for personalization and preference learning.

156M page visits 10.2M searches 5.7M add-to-cart 1.8M purchases 1.26M products

Personalized Search Engine

The interactive local app lets you assume a validation-set user role, then reranks products using the selected user's inferred category affinity, price profile, recent query tokens, and product interaction history.

MLflow Experiment Tracking

MLflow is the experiment ledger for model comparisons: parameters, ranking metrics, artifacts, and model files. The local stack exposes MLflow on port 5002.

Local Startup

Portfolio Hosting Recommendation

For a permanent public portfolio, use a static GitHub Pages demo plus DagsHub-hosted MLflow if you want public experiment tracking. Keeping the MLflow server local avoids paying to keep infrastructure warm.

ClickHouse Analytical Store

ClickHouse stores event data, user features, predictions, and evaluation outputs in the local production-like pipeline. It is intentionally local-first because the public demo does not need arbitrary database queries.

Open local ClickHouse UI ClickHouse docs

Local Startup

make start
.venv/bin/recsys schema apply
open "http://localhost:8124/play?theme=dark"

Why Not Host It Here?

GitHub Pages cannot run databases. A hosted ClickHouse service is useful for real analytics workloads, but for a small portfolio demo it is simpler and more robust to publish static artifacts or use a tiny DuckDB/SQLite backend elsewhere.

Architecture

Raw data

RecSys parquet event files and target labels are stored locally under data/raw.

Processing

Polars builds user/item features and static demo artifacts from event history.

Storage

ClickHouse is the analytical feature/evaluation store for local reproducible runs.

Tracking

MLflow records model parameters, metrics, and artifacts for each experiment.

Serving

FastAPI serves the local personalized search UI and ranking endpoints.

Publishing

GitHub Pages hosts this static project hub and can later host precomputed demo JSON.

Roadmap

From-scratch models

Popularity, item-item co-occurrence, matrix factorization, and BPR trained/evaluated on validation targets.

Library baselines

LightGBM propensity ranker and optional neural embedding models tracked through MLflow.

Dataset: Synerise RecSys Challenge 2025. The dataset anonymizes user identities, product names, and query text.
